## 운영 규칙

1. 실제 구현과 불일치가 생기면 `API_CURRENT.md`를 우선 수정합니다.
2. 확장 논의는 `API_PLANNED.md`에 기록하고, 구현 완료 시 현재 스펙으로 이동합니다.
